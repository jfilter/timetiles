# Configuration

Environment variables and admin settings for TimeTiles.

## Required Variables

```bash
DATABASE_URL=postgresql://user:password@localhost:5432/timetiles
PAYLOAD_SECRET=your-secret-key-minimum-32-characters  # openssl rand -base64 32
```

In production, also set:

```bash
NODE_ENV=production
NEXT_PUBLIC_APP_URL=https://your-domain.com
LOG_LEVEL=info    # debug, info, warn, error
```

## Geocoding

TimeTiles geocodes text addresses to coordinates using multiple providers with automatic fallback. Configure providers in the admin dashboard at `/dashboard/collections/geocoding-providers`.

| Provider    | API Key Required | Notes                                                   |
| ----------- | ---------------- | ------------------------------------------------------- |
| Nominatim   | No               | Free, based on OpenStreetMap. Enabled by default        |
| Google Maps | Yes              | Most accurate. Configure key in dashboard               |
| OpenCage    | Yes              | Good international coverage. Configure key in dashboard |

Providers are tried in priority order (lower number = higher priority). If one fails, the next is tried automatically.

**Global geocoding settings** at `/dashboard/globals/settings`:

- Enable/disable geocoding globally
- Toggle provider fallback
- Choose selection strategy (priority-based or tag-based)
- Configure cache TTL

## File Uploads

```bash
MAX_FILE_SIZE_AUTHENTICATED=104857600      # 100 MB
MAX_FILE_SIZE_UNAUTHENTICATED=10485760     # 10 MB
UPLOAD_DIR=uploads                         # Base directory for uploads
```

## Rate Limiting

Rate limits for unauthenticated users (authenticated users have per-trust-level limits — see [Usage Limits](./usage-limits)):

```bash
RATE_LIMIT_FILE_UPLOAD=5          # per hour
RATE_LIMIT_PROGRESS_CHECK=100     # per hour
RATE_LIMIT_API_GENERAL=50         # per hour
RATE_LIMIT_WINDOW_MS=3600000      # 1 hour
```

## Batch Processing

Tune performance vs. memory usage:

```bash
BATCH_SIZE_DUPLICATE_ANALYSIS=5000    # Memory efficient for duplicate detection
BATCH_SIZE_SCHEMA_DETECTION=10000     # Larger batches for schema building
BATCH_SIZE_EVENT_CREATION=1000        # Smaller to avoid transaction timeouts
```

## Caching

```bash
CACHE_BACKEND=filesystem           # memory or filesystem
CACHE_DEFAULT_TTL=3600             # seconds (1 hour)
CACHE_MAX_SIZE_MB=500
CACHE_DIR=.cache

# Geocoding cache cleanup
GEOCODING_CACHE_CLEANUP_DAYS=90
GEOCODING_CACHE_MIN_HITS=3

# URL fetch cache (for scheduled imports)
URL_FETCH_CACHE_DIR=/tmp/url-fetch-cache
URL_FETCH_CACHE_MAX_SIZE=104857600          # 100 MB
URL_FETCH_CACHE_TTL=3600                    # seconds
URL_FETCH_CACHE_RESPECT_CACHE_CONTROL=true  # honor RFC 7234 headers
```

## Feature Flags

Configure in the admin dashboard at `/dashboard/globals/settings` under "Feature Flags":

| Flag                           | Default | Description                                    |
| ------------------------------ | ------- | ---------------------------------------------- |
| Allow Private Ingestion        | On      | Users can create private imports               |
| Enable Scheduled Ingestion     | On      | Automated URL-based imports on a cron schedule |
| Enable Public Registration     | On      | New users can self-register                    |
| Enable Event Creation          | On      | Events can be created via import or API        |
| Enable Dataset Creation        | On      | Users can create new datasets                  |
| Enable Ingestion Creation      | On      | Users can start new imports                    |
| Enable Scheduled Job Execution | On      | Scheduled imports run automatically            |
| Enable URL Fetch Caching       | On      | Cache URL fetches to reduce external requests  |

## Dashboard

The admin dashboard at `/dashboard` provides a web interface for managing data. Access levels:

| Role   | Permissions                                           |
| ------ | ----------------------------------------------------- |
| User   | Own catalogs, datasets, and imports                   |
| Editor | User permissions + branding, advanced dataset config  |
| Admin  | Full access: user management, settings, feature flags |

## Content Management

### Pages

Create custom pages with a block-based builder at `/dashboard/collections/pages`. Available blocks: Hero, Features, Stats, Timeline, Testimonials, Rich Text, Call to Action, Newsletter. Pages are accessible at `/{slug}`.

### Navigation

- **Main Menu** (`/dashboard/globals/main-menu`) — up to 6 navigation links
- **Footer** (`/dashboard/globals/footer`) — tagline, social links, link columns, copyright

### Branding

Customize at `/dashboard/globals/branding`:

- Site name and description (browser tab, SEO)
- Logos for light and dark themes
- Favicon (auto-generated from source image)

### Newsletter

Configure the backend service URL in Settings (`/dashboard/globals/settings`) under "Newsletter Configuration". Newsletter signup forms can be added to the footer or pages.

## Scheduled Imports

Users can create automated import schedules that fetch data from a URL on a cron schedule.

**Authentication options** for protected URLs:

- None (public URLs)
- API Key (header or query parameter)
- Bearer Token
- Basic Auth

**Management**: View schedules at `/dashboard/collections/scheduled-ingests`. Check schedule health via `GET /api/admin/schedule-service`. Stuck jobs are cleaned up automatically by a background job.

## Audit Log

TimeTiles tracks user and admin actions for accountability. View logs at `/dashboard/collections/audit-log`.

**What's tracked** — 20 action types across 5 domains: account (login, registration, email/password changes, deletion), admin (role changes, trust levels, feature flags), data (catalog/dataset/event CRUD), import (start, complete, fail), system (settings, geocoding provider changes).

**Privacy** — IP addresses are stored for 30 days, then cleared automatically. The `performedBy` field is only set when the acting user differs from the target user.

## User Data Management

- **Data Export**: Users download all their data as a ZIP archive from Account Settings
- **Account Deletion**: 7-day grace period. Private data is permanently deleted; public data is transferred to an admin account

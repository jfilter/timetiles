# Data Ingestion

Data ingestion is the process of bringing external data into TimeTiles. Upload a file, paste a URL, or configure a recurring schedule. TimeTiles detects the schema, geocodes addresses, and creates events.

## Supported Formats

| Format       | Extensions      | Notes                                                     |
| ------------ | --------------- | --------------------------------------------------------- |
| CSV          | `.csv`          | Comma, semicolon, or tab delimited — auto-detected        |
| Excel        | `.xlsx`, `.xls` | Multi-sheet files create one dataset per sheet            |
| OpenDocument | `.ods`          | Spreadsheet format                                        |
| JSON API     | URL             | Fetched from a remote endpoint, converted to tabular data |

## The Pipeline

Every ingestion — whether a one-time file upload or a recurring schedule — follows the same stages:

```
Upload or fetch from URL
  → Detect sheets and datasets
    → Analyze for duplicates
      → Detect schema (field types, dates, locations)
        → You review and approve the schema
          → Geocode addresses to coordinates
            → Create events
```

### Schema Detection

TimeTiles analyzes your data to identify:

- **Field types** — strings, numbers, dates, booleans
- **Date fields** — which column holds the event timestamp
- **Location fields** — addresses, coordinate pairs, or separate lat/lng columns
- **Enum fields** — columns with a small number of unique values (used for filters)

You review and adjust the detected mappings before processing begins.

### Deduplication

When importing into a dataset that already has events, TimeTiles compares incoming rows against existing data. Deduplication strategies:

- **External ID** — use a unique identifier field from your data
- **Computed hash** — hash a combination of fields (e.g., title + date + location)
- **Content hash** — hash the entire row

### Geocoding

Text addresses are converted to geographic coordinates using multiple providers with automatic fallback:

| Provider    | Notes                             |
| ----------- | --------------------------------- |
| Nominatim   | Free, based on OpenStreetMap      |
| Google Maps | Paid, highest accuracy            |
| OpenCage    | Paid, good international coverage |

Results are cached to minimize API calls and cost. If your data already contains latitude and longitude columns, geocoding is skipped.

### Event Creation

Validated, deduplicated, and geocoded rows become events in the dataset. Each event stores:

- Geographic coordinates (for the map)
- A timestamp (for the timeline)
- All original fields from your data (for display and filtering)

## One-Time Import

1. Go to the **Import** page
2. Choose or create a catalog and dataset
3. Upload your file or paste a URL
4. Review the detected schema and field mappings
5. Click **Start** and watch the progress tracker

## Scheduled Import

Scheduled imports fetch data from a URL automatically — hourly, daily, weekly, or on a custom cron expression.

### Setup

1. In the Import wizard, choose **URL** as your data source
2. Configure the schema and field mappings as usual
3. In the **Schedule** step, choose **Repeat on schedule**
4. Set the frequency (e.g., every 6 hours, daily at midnight)
5. Optionally add authentication headers for protected APIs
6. Save

### How It Works

A background job checks for due schedules every minute. When a schedule fires, it fetches the URL and runs the full pipeline — same as a manual import. If the source schema has changed, you may be asked to approve the new mapping.

### Managing Schedules

From the **Scheduled Imports** section in your account:

- Pause or resume a schedule
- Trigger an immediate run
- View the history of past runs
- Delete a schedule

## Schema Evolution

As your data changes over time, TimeTiles tracks schema versions:

- **New optional fields** — auto-approved, existing events are unaffected
- **Type changes or removed fields** — require your approval before processing continues

## Monitoring Progress

During ingestion, you can track:

- Current stage (schema detection, geocoding, event creation)
- Rows processed vs. total
- Errors encountered (with details)

## Troubleshooting

| Issue                  | Solution                                          |
| ---------------------- | ------------------------------------------------- |
| Import stuck           | Check that background workers are running         |
| Schema approval needed | Review and approve in the admin dashboard         |
| Geocoding failures     | Verify API keys and address formatting            |
| Duplicate events       | Configure a deduplication strategy on the dataset |
| Large file timeout     | Split into smaller files or increase batch size   |

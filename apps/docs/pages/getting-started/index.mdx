# Getting Started with TimeTiles

TimeTiles is a comprehensive spatial data management platform built with modern web technologies. This guide will help you get up and running quickly.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (version 24 or higher)
- **pnpm** (version 10.12.4 or higher)
- **Docker** and **Docker Compose**
- **Git**

## System Requirements

- **Operating System**: macOS, Linux, or Windows with WSL2
- **Memory**: 8GB RAM minimum, 16GB recommended
- **Storage**: 5GB free space for development environment

## What You'll Build

By following this guide, you'll have:

- A fully functional TimeTiles development environment
- PostgreSQL database with PostGIS extensions
- Payload CMS admin interface
- Next.js frontend application
- Comprehensive seed data for testing

## Architecture Overview

TimeTiles uses a monorepo structure with the following components:

```
timetiles/
├── apps/
│   ├── web/          # Main Next.js application
│   └── docs/         # Documentation site (this site!)
├── packages/
│   ├── ui/           # Shared UI components
│   ├── eslint-config/    # Shared ESLint configuration
│   └── typescript-config/ # Shared TypeScript configuration
└── scripts/          # Setup and utility scripts
```

### Technology Stack

- **Frontend**: Next.js 15, React 19, TypeScript
- **CMS**: Payload CMS with PostgreSQL adapter
- **Database**: PostgreSQL 17 with PostGIS extension
- **UI**: shadcn/ui, Tailwind CSS, Lucide React
- **Package Manager**: pnpm
- **Build Tool**: Turborepo
- **Development**: Docker Compose for local services

# Paths kept out of the VM's copy of the monorepo (see run-vm.sh).
#
# The VM syncs from a read-only mount of the host checkout into
# /opt/timetiles-src, which is what bootstrap installs from and what
# Dockerfile.prod uses as its build context. Two reasons things are listed
# here: they are host build artifacts that must not leak into a Linux image
# (node_modules is darwin/arm64), or they are operator state the VM generates
# itself and must survive the sync.
#
# rsync --delete does not remove excluded paths on the receiver, so anything
# listed here also persists across runs inside the VM.
